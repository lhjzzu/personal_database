# 清单笔记
- 价值: 更简洁易用的笔记方式
	- 简洁：字少、轻松撰写和阅读
	- 易用：抓逻辑主线，易阅读
- 关键：逻辑主线
- 为什么推荐清单笔记，而不是思维导图？
	- 清单笔记清晰易读，思维导图篇幅难以控制
- 操作指南
	- 使用`Tab`和 `shif + Tab`来控制层级
- 记录清单笔记的场景
	- 学习过程中
		- 阅读/听课中
		- 重点关注逻辑主线
			- 第一...
			- 第二...
	- 学习之后
		- 凭理解和记忆整理笔记
		- 重点关注逻辑主线
- 清单笔记为何是有效的？
	- 知识的三维结构
		- 分享者所传递的知识，通常包含
			- 逻辑推演
			- 包含关系
				- 第一，第二
				- 组合概念和案例
		- 完整的知识
			- 逻辑主线
			- 知识点层层嵌套
		- 书和课程的知识通常是立体化，系统化的
		- 阅读的过程通常是线性的，一维/二维的
			- 难以还原三维的(立体的)知识结构
			- 难以理解体系化的推演和包含关系
	- 有效的解决办法
		- 用清单笔记，厘清逻辑主线，还原三维结构
- 实践的方法
	- 以主线为核心
	- 以知识点作为补充

# 知识管理
- 知识的定义
	- 数据
		- 孤立无意义的内容
	- 信息
		- 被赋予意义的数据
	- 知识
		- 经过学习理解后的信息
	- 深度知识
		- 形成理解后输出的知识
- <font color='red'>未经理解和学习的资料，只是信息，而不是知识</font>
- 为什么进行知识管理?
	- 信息太多
	- 碎片化->片面->绝对化
- 如何进行管理？
	- 信息->知识->深度知识
		- 学习: 信息->知识
			- 摘录
			- 清单笔记
			- ...
		- 输出: 知识->深度知识
			- 双链
			- 片段化
			- ...
	- 最终让知识规范化和系统化
- 知识的分层管理
	- 信息: 轻度管理
		- 文件夹
		- 网盘
	- 知识: 中度管理
		- 提炼重点
		- 记录清单笔记
	- 深度知识: 重度管理
		- 双链
		- 输出
	- 建立
		- 信息库
		- 知识库
		- 深度知识库
# 为什么输出和写作？
- 输出的作用
	- 学习的角度
		- 整理知识
		- 发现新知识
	- 分享的角度
		- 让更多的人关注
	- 知识管理的角度
		- 让知识体系化
- 输出的场景
	- 清单笔记，整理学习成功
	- 跟踪日志，记录日常
	- 长文，对外分享
	- ...

# 输出的流程
- 获取知识
	- 收集信息
	- 学习知识
	- 记录清单笔记
- 写作
	- 收集素材
		- 大量的素材，可以使用看板工具
		- 随时记录素材，灵感等
	- 厘清主线
		- 清单笔记
	- 回想构思
		- 运用放松时间
		- 思考: 这篇文章怎么写
	- 输出成文
		- 工具: obsidian
		- 配合: markdown
		- 配合: 番茄工作法
			- 在一段时间内，专注于该项任务，破除拖延症
